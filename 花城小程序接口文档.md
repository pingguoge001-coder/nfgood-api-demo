# nfapi 小程序 API 文档总结

这是一个名为 **nfapi** 的开放平台API文档（属于"花城农夫"小程序），托管在 Apizza 平台上。基础接口地址为 `https://open.nfgood.com/api/v1/` + 方法名。整个文档共包含 **14个接口**，分为三大模块：

---

## 一、通用规则

**调用规范：** 所有接口采用 HTTPS 传输、POST 方法提交，数据格式为 JSON，字符编码 UTF-8，签名算法为 MD5。

**公共 Header 参数（每个接口都需要）：**
- `partnerId`：合作者（店铺）ID，开通开放能力时获得
- `nonceStr`：16位随机字符串
- `timestamp`：请求时间戳（单位秒）
- `sign`：签名字符串
- `Content-Type`：application/json

**签名算法：** 拼接字符串格式为 `method={方法名}&nonceStr={nonceStr}&partnerId={partnerId}&timestamp={timestamp}&{请求参数的json字符串}&{secret}`，然后对拼接后的字符串进行 MD5 编码并转大写。

**返回格式：** JSON，正确时返回 `data` 字段，错误时返回 `errors` 数组（含 `message` 和 `locations`）。

---

## 二、模块一：三方平台添加商品

### 1. 笔记转商品（addGoods）
- **URL：** `POST /api/v1/addGoods`
- **功能：** 用户跳转到接口返回的链接后自动关联对应商品
- **请求参数：** name（商品标题）、logo（商品LOGO）、contents（图文内容数组，支持 IMAGE/VIDEO/CONTENT 三种类型）、specs（规格信息，含 name/logoUrl/retailFee零售价/supplyFee供货价/props属性），价格单位为"分"
- **返回：** `noteToGoods` 字段，为小程序跳转链接，可通过 `wx.navigateToMiniProgram` 跳转

## 三、模块二：主动推送（测试中）

该功能尚在测试调试阶段，需联系客服获得 API key 和密钥，需提供公司 logo、名称、接收通知的 URL。

**推送协议：** 系统通过 POST 方式向第三方推送 JSON 数据，收到后需返回 "success" 字符串，否则系统会按一定规则和间隔继续推送（最多10次）。

**推送数据格式：** 包含 orderId（订单ID）、groupKey（帮会API key）、changeType（变更类型：1-新订单、2-售后申请或状态变更、3-订单地址修改、4-发货信息变更）。

### 2. 关联合作商（setRelationErp）
- **URL：** `POST /api/v1/setRelationErp`
- **功能：** 设置 ERP 合作商与帮会的关联关系，关联后才能收到帮会的订单通知
- **请求参数：** erpId（ERP合作商ID，联系客服获取）
- **说明：** 使用帮会 API key 和 secret 调用，帮会重置 key/secret 后需重新绑定

## 四、模块三：api文档（核心业务接口）

### 3. 获取订单列表（listOrders）
- **URL：** `POST /api/v1/listOrders`
- **请求参数：** skip（跳过条数）、limit（获取条数）、ems（发货状态：ALL/NOSEND/SEND）、supplyType（发货类型：ALL/MINE由我发货/OTHER他人发货）、lastTime（获取指定时间以及之后更新的订单）、orderId（订单ID，选填）、updateEndTime（获取指定时间以及之前更新的订单，lastTime大于0时有效）
- **返回：** 含 count（总数）和 listOrders 数组，每个订单包含丰富字段：地址信息（address）、发货信息（listCPInfo，含快递单号/快递公司编码/名称/时间）、售后信息（customInfo，含申请时间/完成时间/金额/原因/状态/拒绝原因）、商品信息（goodsinfo）、规格信息（specInfo，含规格属性和属性构成）、寄件人信息（sendInfo）、自提相关（takeType/takeState/takeTime）、支付信息（payFee/payState/payPerPrice/paymentFee）、用户下单扩展信息（tplExts）等

### 4. 获取售后订单（refundOrders）
- **URL：** `POST /api/v1/refundOrders`
- **请求参数：** skip（跳过条数）、limit（获取条数）、key（搜索关键字）、customType（第一次传 REFUND，后面根据结构返回来传）

### 5. 获取商品列表（listGoods）
- **URL：** `POST /api/v1/listGoods`
- **请求参数：** under（上下架状态：ALL/UNDER下架的/SELL在售的）、source（产品来源：ALL/SELF自己上架的/BIND绑定的/BIND_SIFT从农夫精选上架/AUTHER授权给我的/SELF_AUTHER授权给我且自己上架的/UNKNOW未知）、key（搜索key，选填）、skip（跳过条数）、limit（获取条数）

### 6. 订单发货（orderDelivery）
- **URL：** `POST /api/v1/orderDelivery`
- **请求参数：** id（单号）、cp（快递公司编码）、code（快递编号）、oldCode（要修改的单号，为空则表示新增）

### 7. 快递公司编码（参考文档）
- 提供了所有支持的快递公司编码对照表，包括：SF（顺丰）、YTO（圆通）、ZTO（中通）、STO（申通）、HTKY（百世汇通）、YUNDA（韵达）、JD（京东）、JTSD（极兔速递）、ZJS（宅急送）、GTO（国通）、HHTT（天天）、FAST（快捷）、EMS、POSTB（邮政小包）、YZPY（邮政快递）、UC（优速）、DBKD（德邦）、JIUYE（九曳供应链）、ZHONGYOU（众邮）、ANE（安能）、PJ（品骏）、NONE（自定义）

### 8. 更新商品库存（updateGoodsInventory）
- **URL：** `POST /api/v1/updateGoodsInventory`
- **请求参数：** goodsId（商品ID，平台商品ID）、specCode（规格编码，支持平台规格ID和二方Code）、propCode（属性编码，支持平台属性Code和二方编码，如果规格包含属性则此字段必填）、optType（0追加/1修改）、num（追加或者修改的数量）

### 9. 获取商品详情（goodsInfo）
- **URL：** `POST /api/v1/goodsInfo`
- **请求参数：** goodsId（商品ID）
- **返回：** 包含商品详细信息，如 availableBegin/availableEnd（商品开售/停售时间）等完整商品数据

### 10. 获取订单详情（orderInfo）
- **URL：** `POST /api/v1/orderInfo`
- **请求参数：** orderId（订单ID）
- **返回：** 返回字段与订单列表相同

### 11. 同步商品到草稿箱（syncGoods）
- **URL：** `POST /api/v1/syncGoods`
- **请求参数：** 与"笔记转商品"相同，包含 name、logo、contents、specs 等完整商品信息
- **功能：** 将商品同步到草稿箱

## 五、测试信息

文档提供了测试用的 partnerId 和 secret，测试环境不校验 timestamp，方便开发者调试。

总体来看，这是一套围绕"花城农夫"小程序的电商开放平台API，涵盖了**商品管理（添加/同步/查询/库存更新）、订单管理（列表/详情/发货/售后）、ERP对接（关联合作商/主动推送）**三大核心业务场景，共14个接口。
